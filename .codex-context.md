# Codex Context: Kyma-ter & Kyma Agent

## Tổng quan hệ sinh thái

Kyma API (kymaapi.com) là một AI API gateway — users tạo 1 account, access 20+ LLM models qua 1 balance. Có 2 products chính:

### 1. Kyma Agent (`kyma` command)
- **Repo**: kyma-api (monorepo) → `packages/kyma-agent/`
- **Package**: `@kyma-api/agent` trên npm (v0.1.5)
- **Tech**: TypeScript, Pi SDK, ESM
- **Chức năng**: Terminal-first AI coding agent, giống Claude Code / Cursor nhưng dùng Kyma API backend
- **Features**: multi-model (fast/coding/reasoning/cheap/vision/long modes), OAuth login, per-project KYMA.md config, session cost tracking
- **Commands**: `/login`, `/models`, `/mode`, `/balance`, `/status`, `/doctor`, `/usage`

### 2. Kyma-ter (`kyma-ter` command)
- **Repo**: kyma-ter (standalone, private, Go + React)
- **Distributed via**: cùng package `@kyma-api/agent` (postinstall downloads Go binary)
- **Tech**: Go backend (Cobra CLI, Gorilla Mux, SQLite, WebSocket, PTY), React 19 frontend (Vite, Zustand, TypeScript)
- **Chức năng**: Multi-agent terminal workspace — chạy nhiều AI agents song song trong browser UI
- **Features**:
  - Tiling pane layout (2-16 panes, auto-arrange hoặc preset grid)
  - Keyboard shortcuts: Ctrl+K (new Kyma Agent), Ctrl+N (new terminal), Ctrl+W (close), Ctrl+T (new workspace), Ctrl+A (agent workspace), Ctrl+, (settings)
  - Drag & drop file upload vào terminal panes
  - Agent types: Kyma Agent (command: `kyma`, color: #eab308) và Shell
  - Settings UI: Agent Workspace config + Keyboard Shortcuts
  - Background auto-update: check `ter-latest.txt` → download new binary silently
  - Local-first: mọi thứ chạy trên localhost:18800, SQLite local, zero cloud dependency
  - WebSocket-based terminal I/O, session heartbeat
  - Tasks side panel, inline note creation

### Cách 2 products liên kết
```
curl -fsSL https://kymaapi.com/install.sh | bash
  → detects OS
  → installs Node.js if needed
  → npm install -g @kyma-api/agent
  → postinstall downloads kyma-ter Go binary → ~/.kyma/ter/bin/
  ├── kyma        → AI coding agent CLI (Node.js, built-in)
  └── kyma-ter    → terminal workspace UI (Go binary, downloaded on postinstall)

# Alternative: npm directly
npm install -g @kyma-api/agent
```

Khi user chạy `kyma-ter`, Go server start → auto-open browser → workspace UI. Trong mỗi pane, user có thể spawn Kyma Agent (chạy `kyma` command) hoặc Shell.

## Architecture

### kyma-ter repo structure
```
cmd/kyma-ter/main.go          # CLI entry (Cobra), version injection, auto-updater
internal/
  config/config.go             # ~/.config/kyma-ter/config.json, agent definitions
  server/server.go             # HTTP server, REST API, WebSocket endpoints
  db/                          # SQLite — tasks, locks, messages
  ptymanager/                  # PTY session management (creack/pty)
  updater/updater.go           # Background version check + binary download
  web/                         # Embedded React SPA (go:embed)
  wsbridge/                    # WebSocket ↔ PTY bridge
frontend/src/
  store/ui.ts                  # Zustand — tabs, panes, layout, focus
  store/sessions.ts            # Session CRUD via API
  store/settings.ts            # Keyboard shortcuts, persist v3
  hooks/useKeyboardShortcuts.ts
  components/Settings/SettingsPage.tsx
  components/Terminal/Terminal.tsx
  components/AgentWorkspace/AgentWorkspaceModal.tsx
  utils/spawn.ts               # spawnAgent, spawnShell, spawnKymaIfReady
  types.ts                     # AgentConfig, DEFAULT_AGENTS
  api/client.ts                # REST + WebSocket client
```

### kyma-agent package structure (trong kyma-api monorepo)
```
packages/kyma-agent/
  bin/kyma.js                  # Entry → Pi SDK launcher
  bin/kyma-ter.js              # Wrapper → runs Go binary from ~/.kyma/ter/bin/
  scripts/postinstall.js       # Pi symlinks + kyma-ter binary download
  src/sdk/
    main.ts                    # Arg parsing, bootstrap, onboarding, session
    extension.ts               # Provider factory (OAuth, models) + Runtime factory (commands, branding)
    auth.ts                    # OAuth device flow, credential storage
    api.ts                     # HTTP client, cost formatting
    session.ts                 # Pi session creation
    onboarding.ts              # First-run setup, login prompt
  src/models.ts                # 13+ curated models with costs, context, compat
  themes/kyma-dark.json        # Custom dark theme (#C9A84C gold accent)
```

### Distribution & Update flow
```
Release kyma-ter:
  1. kyma-ter repo: make release-binaries VERSION=x.y.z
  2. gh release create ter-vx.y.z ... --repo sonpiaz/kyma-releases
  3. Update ter-latest.txt in kyma-releases repo
  4. Update kymaTerminal field in kyma-api packages/kyma-agent/package.json
  5. npm publish @kyma-api/agent

Auto-update (client-side):
  1. kyma-ter startup → background goroutine
  2. Fetch https://raw.githubusercontent.com/sonpiaz/kyma-releases/main/ter-latest.txt
  3. Compare with compiled Version
  4. If newer → download to ~/.kyma/ter/bin/kyma-ter.new
  5. Next startup → ApplyPending() swaps .new → current
```

### Data paths
```
~/.kyma/
  agent/                       # Kyma CLI config
    auth.json                  # OAuth tokens (0600)
    settings.json              # Default model, theme, thinking level
    themes/kyma-dark.json
  ter/                         # kyma-ter binary
    bin/kyma-ter               # Go binary
    version                    # "0.1.0"

~/.config/kyma-ter/
  config.json                  # Server config (port, bind, agents, API keys)
  *.db                         # SQLite databases
```
